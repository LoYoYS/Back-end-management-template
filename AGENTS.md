# Project Instructions

本文件为当前项目目录的编码遵守规则，适用于本目录及其子目录。

## Code Design

- 避免过度封装；仅在明显提升可读性、复用性或健壮性时进行适度封装。
- 优先采用高效、简洁、直接的实现方式，避免为简单逻辑引入不必要的抽象层。
- 在满足可读性、可维护性和健壮性的前提下，优先选择性能更高、代码更短的实现方案。

## Implementation

- 实现功能时优先使用最简洁的方法，避免冗余分支、重复代码和无意义的中间变量。
- 不为“看起来规范”而增加额外包装代码；每一层结构、函数、组件和状态都应有明确价值。
- 代码应保持稳定、可用、易懂，不能因追求短小而牺牲边界处理和错误处理。

## Comments

- 重点函数、关键方法、复杂判断、核心数据转换逻辑应添加简明注释。
- 注释始终使用中文。
- 注释应解释设计意图、边界条件或关键行为，不写显而易见的废话注释。

## Vue Style Rules

- 编写 `.vue` 文件样式时，始终使用 `scss`。
- 样式选择器优先多层级嵌套，便于样式面板折叠和维护。
- 类名命名应使用 `kebab-case`。

## Documentation

- 编写文档时始终以中文为主。
